QuackPlan User Manual

Well Planning Software — Reference Guide for Drilling Engineers

Overview

QuackPlan is a browser-based well planning tool that covers trajectory design, well schematic definition, torque & drag analysis, hydraulics, and AFE cost estimation. All data is stored locally in your browser — no server or internet connection is required during use.

ModuleWhat it does
TrajectoryDefine or build a wellbore path; apply tortuosity
Well SchematicDefine casing, liner, and open-hole intervals; generates live visual
Casing / BHADefine drillstring components, bit nozzles, and MWD pressure losses
Drilling FluidSet mud properties and rheology model
Activity PlanBuild a day-by-day activity schedule with service costs
Trajectory PlotVertical section + plan view of the wellbore
Torque & DragTorque, buckling, overpull, and broomstick charts with FF sensitivity
HydraulicsSPP vs flow rate sweep, ECD, HSI, and pressure breakdown
Initial AFECumulative days and cost vs depth chart
CompareSide-by-side comparison of up to three scenarios

Typical Workflow

  1. Create a project. Click + Project in the header or the hierarchy panel. Then add a Field/Pad, Well, Borehole, and Scenario by right-clicking each level in the tree.
  2. Select a scenario. Click a Scenario in the tree. Its name appears in the header context label. All inputs and outputs belong to the active scenario.
  3. Enter trajectory. Go to the Trajectory input tab. Use Option 1 to paste a survey or Option 2 to define build/hold criteria. Optionally add tortuosity.
  4. Define the well schematic. Switch to Well Schematic and add casing and liner strings. The visual schematic on the right updates live.
  5. Define the BHA. Switch to Casing / BHA. Enter drillstring components bottom-to-top (bit first). Set bit nozzles and MWD/drive system pressure losses.
  6. Set drilling fluid. Switch to Drilling Fluid. Choose a rheology model and enter mud properties.
  7. Build the activity plan. Switch to Activity. Add activities, service day rates, lump sums, and casing costs.
  8. Compute. Click ⚡ Compute. The engine runs torque & drag, buckling, hydraulics, and AFE calculations.
  9. Review outputs. Click any tab in the footer to view charts. Use the chart interaction tools to read values, freeze scenarios, and add annotations.
  10. Compare scenarios. Switch to the Compare input tab to run a side-by-side comparison of up to three scenarios.

Project Hierarchy

All data is organized in a five-level tree:

Project

    └── Well

Managing the tree

  • Add: Click + next to any node, or the + Project button in the header.
  • Rename: Right-click a node and choose Rename.
  • Delete: Right-click a node and choose Delete. Deleting a parent removes all children.
  • Select: Click any Scenario to make it active. The header shows the well and scenario name.
Data persistence All data is saved automatically to your browser's local storage. Clearing browser data will erase projects. Export important scenarios before clearing.

Trajectory Input

The Trajectory tab has three sub-tabs: Option 1, Option 2, and Tortuosity. Both options produce the same internal survey (minimum curvature) — choose whichever matches your data source.

Option 1 — Paste Survey

Direct entry of measured depth, inclination, and azimuth. Designed for importing an existing survey or Landmark/Halliburton report.

ColumnUnitsInput or Calculated
MDftInput
Inc°Input
Azi°Input
TVDftCalculated
TVDssftCalculated
DLS°/100'Calculated
Paste from Excel Copy three columns (MD, Inc, Azi) from any spreadsheet and paste directly into the table. QuackPlan auto-fills all rows. The first station should always be MD=0, Inc=0, Azi=0.

Click + Add station to append a row manually. Delete a row with the × button at the right.

Option 2 — Mixed Criteria

Build a trajectory by specifying what is known at each design point rather than entering a full survey. Each row has a Define selector that controls which columns are inputs and which are solved.

Define modeInputs (yellow)Solved (blue)
MD / Inc / AziMD, Inc, AziTVD, DLS
Inc / Azi / TVDInc, Azi, TVDMD, DLS
Inc / Azi / DLSInc, Azi, DLSMD, TVD

This is ideal for designing a well profile from target depth and inclination requirements, then solving for the required MD or DLS.

Tortuosity

Both trajectory options support a tortuosity overlay that adds realistic micro-doglegs to simulate actual wellbore roughness. This affects torque & drag results.

FieldDescription
Start MD / End MDDepth interval over which tortuosity is applied
Tortuosity (°/100')Magnitude of added random dogleg. 0.5–1.5 is typical; above 3 is severe.
ModeRandom Inc Azi — independently randomizes inclination and azimuth perturbations
When to use tortuosity Always add tortuosity to planned trajectories before running torque & drag. A smooth planned path underestimates real friction. Typical values: 0.5 °/100' for casing sections, 1.0–2.0 °/100' for rotary-steerable openhole sections.

Well Schematic Input

Define the casing and liner program. The visual schematic on the right side of the screen updates immediately as you add or modify rows.

FieldDescription
DefinitionString type: Surface Casing, Intermediate Casing, Production Casing, or Liner
Size (in)Nominal OD in inches (e.g. 13.375, 9.625, 7)
MD Top (ft)Top of string. Enter 0 or leave blank for surface strings. For liners, enter the liner top (hanger depth).
MD Bottom (ft)Casing shoe / liner shoe measured depth

Add rows in order from largest (shallowest) to smallest (deepest) string. The schematic draws shoe triangles, casing strings, and liner hangers automatically. Open hole below the last casing shoe is shown as a wavy line.

Schematic feeds hydraulics The annular geometry defined here is used directly in the hydraulics calculation to compute section-by-section annular pressure losses and ECD.

Casing / BHA Input

Define the bottomhole assembly and drillstring components from bit to surface. This data drives both torque & drag and hydraulics calculations.

BHA Components Table

Enter components bottom to top — bit first, then DC, HWDP, DP in sequence.

FieldDescription
ComponentFree-text label (e.g. Bit, Drill Collar, HWDP, Drill Pipe, MWD/LWD)
OD (in)Outside diameter in inches
ID (in)Inside diameter in inches (used for pipe pressure drop)
Weight (lbs)Total component weight in air
Length (ft)Component length in feet

Bit Nozzles

Enter each nozzle size and quantity. QuackPlan calculates the Total Flow Area (TFA) automatically.

FieldDescription
Size (1/32")Nozzle diameter in 1/32" increments (e.g. 12 = 12/32" = 3/8")
# NozzlesNumber of nozzles at this size
TFA (in²)Calculated automatically — sum of all nozzle areas

TFA drives the bit pressure drop calculation: ΔP = ρ·V²/1120, where V is nozzle velocity.

MWD / Drive System Pressure Loss

Enter fixed pressure drops for MWD tools and the drive system (motor or rotary steerable). These are added directly to the total standpipe pressure. Common values:

  • MWD tool: 300–600 psi depending on tool and flow rate
  • PDM drive: 400–900 psi at operating differential
  • RSS drive: 200–500 psi

When Herschel-Bulkley is selected, also enter:

  • τ₀ (yield stress): typically 5–15 lb/100ft² for OBM
  • n (flow index): 0.5–0.9 for typical drilling fluids
  • K (consistency index): fit from rheometer data

Casing Costs

Enter unit cost, length, and cement cost for each casing string. Total casing cost per string is calculated automatically and included in the AFE.

Trajectory Plot Output

Shows two views of the wellbore path side by side. Click Trajectory in the footer after running Compute (or click Redraw any time after entering trajectory data).

Vertical Section (left panel)

  • X axis: Horizontal departure from wellhead (ft) — always positive, direction of maximum departure
  • Y axis: True Vertical Depth (ft), surface at top, TD at bottom
  • Casing shoe triangles are plotted at the TVD of each shoe, with the string size and TVD labeled
  • MD tick marks at regular intervals along the path
  • Blue dot = surface (wellhead), red dot = TD

Plan View (right panel)

  • X axis: Easting displacement from wellhead (ft)
  • Y axis: Northing displacement from wellhead (ft), North up
  • North arrow in the top-right corner of the plot
  • Equal aspect ratio — the wellpath is not distorted
Horizontal departure vs. vertical section Horizontal departure is the straight-line distance from the wellhead to each point in the horizontal plane. For directional wells with azimuth changes, the vertical section (projected on a single reference azimuth) is a different value. QuackPlan currently plots departure. If your well has significant azimuth walk, the VS panel shows the true 2D path shape.

Torque Output

Surface torque vs. measured depth for rotary drilling, computed using the Johancsik soft-string model.

Controls

ControlDescription
WOB (klbs)Weight on bit. Higher WOB increases torque in the BHA and lower string.
FF Low / Mid / HighThree friction factor values plotted simultaneously. Typical range: 0.15–0.25 for WBM cased hole, 0.20–0.35 for OBM openhole.
Rotary checkboxWhen checked, uses rotary mode (lower effective friction). Uncheck for sliding mode.

Interpreting the chart

  • Higher torque at surface = more friction in the string
  • Steps in the torque profile correspond to changes in string geometry (pipe OD transitions, casing shoe depths)
  • If the mid FF curve approaches the rig's top drive torque limit, consider reducing WOB or using a lubricant

Buckling Output

Shows the critical buckling loads and compressive axial load vs. depth, using Dawson-Paslay (sinusoidal) and Chen (helical) criteria.

Chart lines

LineMeaning
Red — SinusoidalCritical load for sinusoidal buckling onset. Pipe buckles sinusoidally above this compressive load.
Blue — HelicalCritical load for helical buckling. More severe — causes lockup and casing wear. Must be avoided during sliding.
Green — Compressive LoadActual axial compressive load in the string. Where this exceeds Sinusoidal, buckling is active.

Controls

  • WOB: Increases compressive load in the BHA
  • FF Low / Mid: Two friction factor sensitivities
Helical buckling during sliding If the compressive load exceeds the helical critical load in a long openhole section, the string will lock up and weight cannot be transferred to the bit. Redesign the BHA or reduce WOB if this occurs.

Overpull Output

Hook load vs. measured depth during pull-out-of-hole (POOH), showing three friction factor sensitivities.

Controls

ControlDescription
Block (kips)Traveling block and top drive weight. Calibration value — set from rig data.
DP weight (ppf)Drillpipe linear weight in pounds per foot. Used to calculate air weight of the string.
MW (ppg)Mud weight for buoyancy correction
FF Low / Mid / HighFriction factor sensitivity range

Interpreting the chart

  • Hook load increases as you pull the string off bottom
  • The spread between FF curves represents the overpull margin uncertainty
  • If the high-FF curve approaches the rig's maximum hook load rating, the well geometry or mud weight needs review
  • Calibrate the model against actual pick-up/slack-off weights from the rig during drilling

Broomstick Output

A composite hook load chart showing all three drillstring operating modes simultaneously at a single friction factor, with depth increasing downward.

LineModeInterpretation
Blue — RIHRunning in holeLower hook load; gravity assists descent
Green — Rot-On (WOB)Rotary drilling at WOBMiddle hook load; used for set-down weight checks
Red — POOHPulling out of holeHighest hook load; friction opposes upward motion

The spread between RIH and POOH lines is the total friction contribution. A wide spread in openhole sections indicates high wellbore friction — consider wipers or optimizing the trajectory.

FF control

Use a single friction factor for all three lines. Adjust to bracket the expected friction range for the mud system and formation type.

Hydraulics Output

Two simultaneous views: an SPP vs. flow rate sweep (left) and a pressure breakdown donut chart (right).

SPP Sweep Chart

  • X axis: Flow rate (gpm)
  • Y axis: Standpipe pressure (psi)
  • The red dashed line is the rig SPP limit (set in Drilling Fluid inputs)
  • The orange dot marks the current operating point (flow rate and calculated SPP)
  • ECD at bit and bit HSI are annotated in the chart header

Interactive Controls

ControlEffect
MW sliderAdjusts mud weight from 8–16 ppg without re-entering fluid inputs. Recalculates SPP and ECD.
Flow Rate sliderMoves the operating point along the curve. Updates ECD and HSI in real time.

Pressure Breakdown Donut

Shows how total SPP is distributed across components:

  • Annular: Pressure loss in all annular sections (largest contributor in deep wells)
  • Pipe: Friction loss inside the drillstring
  • Bit: Pressure drop across nozzles (controls HSI)
  • MWD/DS: Fixed tool pressure losses entered in Casing/BHA

Key metrics

MetricFormulaTarget
ECD at bit (ppg)MW + AnnLoss / (0.052 × TVD)Below fracture gradient
Bit HSI (hhp/in²)Bit ΔP × Q / (1714 × bit area)> 1.0 hhp/in²
Nozzle velocity (ft/s)Q / (3.117 × TFA)250–400 ft/s typical
Warnings QuackPlan flags three conditions automatically: SPP exceeding rig limit, ECD above 13.5 ppg (potential fracture), and HSI below 1.0 (poor hole cleaning). These appear at the bottom of the donut chart.

Initial AFE Output

Cumulative days and cost plotted against depth — the standard "staircase" well cost chart used in AFE submissions.

  • Blue line (left axis): Cumulative planned days vs. depth
  • Green line (right axis): Cumulative planned cost vs. depth
  • Total days and total AFE cost are annotated in the top-left corner

Horizontal steps in the depth axis represent non-drilling activities (rig move, casing, cement wait) where cost accumulates but depth does not advance. The slope of diagonal segments represents drilling rate per dollar.

AFE tip Add lump sum costs (logging, coring, liner hanger) in the Services table rather than the Activity table. They are added to the final total but do not appear as individual steps on the staircase.

Chart Interaction

All output charts support three interactive features accessible via mouse. No toolbar buttons needed.

Crosshair & Tooltip

Hover the mouse over any chart. A dashed crosshair follows your cursor and a tooltip displays the exact values at that point:

  • Torque / Overpull / Buckling / Broomstick: depth (MD ft) and the primary axis value
  • Hydraulics: flow rate (gpm) and SPP (psi)
  • Trajectory VS: TVD (ft) and horizontal departure (ft)
  • Trajectory Plan: Northing (ft) and Easting (ft)
  • AFE: depth (ft) and cumulative days

Freeze Snapshot

Right-click anywhere on a chart and choose ❄ Freeze current curves. The current set of curves is pinned to the chart as a semi-transparent dashed reference. You can then:

  • Change inputs (mud weight, WOB, friction factor) and recompute
  • The frozen curves remain visible for comparison alongside the updated live curves
  • Multiple snapshots can be stacked — each is labeled "Snap 1", "Snap 2", etc.
  • Right-click → Clear N frozen snapshots to remove them
Freeze workflow example Freeze the torque chart at FF=0.25 (expected clean hole). Change FF to 0.40 (stuck pipe scenario) and recompute. The live curves show the worst case while the frozen snap shows the baseline — immediately visible on the same chart.

Annotations

Right-click at a specific point of interest and choose 💬 Add annotation here. Type a label (e.g. "KOP", "Casing shoe", "Max WOB limit"). A red pin and labeled box appear at that data point. Annotations persist through recomputes and can be frozen along with curves.

Right-click → Clear annotations removes all annotations from that chart.

Scenario Compare Input

Compare up to three saved scenarios side by side. Switch to the Compare input tab in the header.

  1. Select Scenario A, B, and optionally C from the dropdown menus
  2. Click ⚡ Compare
  3. A metrics table shows key values (max torque, max hookload, pump pressure, ECD, total days, total cost) for each scenario
  4. A trajectory overlay chart plots the TVD vs. MD path for each scenario in different colors
Comparing across wells You can compare scenarios from different wells or boreholes. Use this to evaluate alternative well profiles, mud types, or BHA designs against each other on a common basis.

Buoyancy Factor

The T&D engine uses BF = 1 − (MW / 65.5) for steel in ppg units. This is applied to pipe weight when computing axial loads and friction forces.

Minimum Curvature Method

QuackPlan uses minimum curvature throughout. The dogleg angle DL is calculated as:

cos(DL) = cos(I₂ − I₁) − sin(I₁)·sin(I₂)·(1 − cos(A₂ − A₁))
RF = 2/DL · tan(DL/2)   [ratio factor; RF = 1 when DL = 0]
ΔN = ΔMD/2 · (sin(I₁)cos(A₁) + sin(I₂)cos(A₂)) · RF
ΔE = ΔMD/2 · (sin(I₁)sin(A₁) + sin(I₂)sin(A₂)) · RF
ΔTVD = ΔMD/2 · (cos(I₁) + cos(I₂)) · RF
